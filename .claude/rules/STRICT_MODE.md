## STRICT MODE

General safety rules that apply to all work:

- Do NOT delete or overwrite files without understanding what they contain
- Do NOT modify test assertions to make failing tests pass — fix the code instead
- Do NOT commit secrets, credentials, or .env files
- Do NOT push to remote without explicit approval
- Do NOT run destructive git operations (force push, reset --hard) without asking
- All existing tests must pass before any PR or commit
- When in doubt, STOP and ASK
